## Why Traditional MFA is Like Installing a Better Lock on a Cardboard Door

Last month, I watched a seasoned CISO explain why their "bulletproof" multi-factor authentication got bypassed in under 6 minutes. The attacker didn't need to crack codes or break encryption—they simply exploited the step-up process itself.

This is the **authentication paradox**: the more steps you add for security, the more friction you create for users. The more friction you create, the more users find workarounds. The more workarounds exist, the more attack vectors you introduce.

Traditional step-up authentication follows this flawed logic:
1. User logs in with password
2. System requests additional verification for sensitive actions
3. User provides SMS code, authenticator token, or biometric
4. System grants elevated access

**The problem?** You're still building on a foundation of passwords—the digital equivalent of quicksand.

### Step 2: User Info Recovery with WalletConnect Session Data

Here's where ZTF gets powerful—the user's cryptographic session info is recovered from the authentication endpoint:

```typescript
// ConfigService.tsx - Extract WalletConnect session from user info
const loadAppConfig = async (keycloak: any) => {
  try {
    const realm = 'ztf_demo';
    const userInfoUrl = `https://auth.solvewithvia.com/auth/realms/${realm}/protocol/openid-connect/userinfo`;

    const response = await fetch(userInfoUrl, {
      method: 'GET',
      headers: {
        'Authorization': `Bearer ${keycloak.token}`,
        'Content-Type': 'application/json'
      }
    });

    const userInfoData = await response.json();

    // Extract and decode walletConnectSessionInfo
    let decodedWalletConnectInfo = null;
    if (userInfoData.walletConnectSessionInfo) {
      try {
        const decodedString = atob(userInfoData.walletConnectSessionInfo);
        decodedWalletConnectInfo = JSON.parse(decodedString);
      } catch (decodeError) {
        console.error('Failed to decode walletConnectSessionInfo:', decodeError);
      }
    }

    setWalletConnectInfo(decodedWalletConnectInfo);
  } catch (err) {
    console.error('Error loading app config:', err);
  }
};
```

### Step 4: The Step-Up Authentication Magic - personal_sign

When the user performs sensitive actions, the system requests cryptographic proof via [personal_sign](https://eips.ethereum.org/EIPS/eip-191) using [WalletConnect v2.0](https://docs.walletconnect.com/):

```typescript
// TransactionService.tsx - The actual step-up authentication
const signMessage = async (message: string): Promise<string | null> => {
  if (!client || !session) {
    console.error('WalletConnect not connected');
    return null;
  }

  try {
    setIsLoading(true);

    // Zero Trust validation - verify active session
    const activeSessions = client.session.getAll();
    const hasActiveSession = activeSessions.some(s => s.topic === session.topic);

    if (!hasActiveSession) {
      throw new Error('Session not found in active sessions. Please connect your wallet again.');
    }

    // Extract account from viasecurechain namespace
    let accounts: string[] = [];
    if (session.namespaces.viasecurechain?.accounts?.length > 0) {
      accounts = session.namespaces.viasecurechain.accounts;
    } else {
      // Fallback: find any namespace with accounts
      for (const [namespace, config] of Object.entries(session.namespaces || {})) {
        if (config.accounts && config.accounts.length > 0) {
          accounts = config.accounts;
          break;
        }
      }
    }

    const fullAccount = accounts[0]; // e.g., "viasecurechain:mainnet:0xbA447B1..."
    const fromAccount = fullAccount.split(':')[2]; // Extract address
    const chainId = fullAccount.split(':').slice(0, 2).join(':');

    // Verify personal_sign method is available
    const availableMethods = session.namespaces.viasecurechain.methods || [];
    if (!availableMethods.includes('personal_sign')) {
      throw new Error(`❌ personal_sign method not available. Available methods: [${availableMethods.join(', ')}]`);
    }

    // Request cryptographic signature for step-up authentication
    const params = [
      ethers.utils.hexlify(ethers.utils.toUtf8Bytes(message)),
      fromAccount
    ];

    const signature = await client.request({
      topic: session.topic,
      chainId: chainId,
      request: {
        method: 'personal_sign',
        params: params
      }
    });

    setLastSignature(signature as string);
    return signature as string;

  } catch (error) {
    console.error('Message signing failed:', error);
    throw error;
  } finally {
    setIsLoading(false);
  }
};
```
